# AI 目标

## 问题

OpenRA 默认 RA AI 在扩张节奏、防守优先级、兵种构成与多线压力上经常偏弱或模式化，导致单人/练习体验偏「摆烂」或可被套路克制。

## 设计原则

1. **不作弊**：不给隐藏资源、视野或建造速度；公平规则下更会决策。
2. **经典节奏**：经济曲线与科技时机贴近原版；避免「瞬间满科技海」。
3. **可分层**：保留接近原版的难度档，另提供 `enhanced-*` 档位。
4. **先 YAML，后 C#**：能用 MiniYaml / 现有 bot 模块解决的，不先上自定义程序集。

## 目标行为（按优先级）

| 优先级 | 行为 | 说明 |
|--------|------|------|
| P0 | 稳定扩张 | 矿车数量、新矿场时机、电厂冗余 |
| P0 | 基础防守 | 前线/基地关键建筑的防空与防坦克 |
| P1 | 兵种构成 | 对坦克/步兵/空军的克制意识（简单计数即可）
| P1 | 骚扰与压力 | 定期小股骚扰，避免只会一路 A 上去 |
| P2 | 多线 / 海空军时机 | 地图类型感知（需更多数据与测试） |
| P2 | 工程师与超级武器意识 | 高风险高回报，后期再做 |

## 非目标（首期）

- 达到顶级人类水平的微观
- 为竞技天梯训练专用神经网络（可做研究分支，不进入默认）
- 通过改单位属性让 AI「显得更强」

## 实现落点

- 覆盖 / 扩展 OpenRA `ra` 的 AI 相关 YAML（见 `mod/ra-enhanced/rules/`）
- 文档化每个 AI 档位的意图与测试地图
- Week 2 交付 v0 基线（见根目录 [ROADMAP.md](../ROADMAP.md)）

## 测试想法

- 固定种子 / 固定地图各打 5 局，记录：首矿场时间、首次进攻时间、是否断电、是否被偷家
- 对比官方默认 AI 与 `enhanced-normal`
